## Mod 来源

SVL 支持从以下来源安装 Mod：

| 来源 | 说明 | 推荐度 |
|------|------|--------|
| **NexusMods** | 最大的 Mod 平台，SVL 深度集成 | ⭐⭐⭐⭐⭐ |
| **CurseForge** | 备用 Mod 平台 | ⭐⭐⭐⭐ |
| **本地文件** | ZIP 压缩包或文件夹 | ⭐⭐⭐ |
| **拖放安装** | 快速安装 | ⭐⭐⭐⭐ |

## 从 NexusMods 安装

### 登录 NexusMods

首先需要登录 NexusMods 账号：

1. 进入 **「设置」** → **「NexusMods」**
2. 点击 **「登录」**
3. 在浏览器中授权
4. 返回 SVL，登录完成

### 搜索 Mod

1. 选择要安装 Mod 的实例
2. 进入 **「自动安装」** 选项卡
3. 在搜索框输入 Mod 名称
4. 按 `Enter` 或点击搜索图标

### 选择 Mod

搜索结果会显示：

- **Mod 名称** - 显示名称
- **作者** - Mod 作者
- **下载量** - 总下载次数
- ** endorsements** - 点赞数

点击 Mod 查看详情。

### 下载安装

1. 在 Mod 详情页，查看版本列表
2. 选择适合你游戏版本的版本
3. 点击 **「下载并安装」**
4. 等待下载完成
5. Mod 会自动安装到当前实例

## 从 ZIP 文件安装

### 方法一：拖放安装

1. 下载 Mod 的 ZIP 文件
2. 将 ZIP 文件直接拖放到 SVL 窗口
3. 选择目标实例
4. 等待安装完成

### 方法二：菜单安装

1. 进入实例设置 → **「Mod 管理」**
2. 点击 **「从 ZIP 安装」**
3. 选择 ZIP 文件
4. 点击 **「打开」**
5. 等待安装完成

### ZIP 文件要求

正确的 Mod ZIP 文件应包含：

```
mod.zip
├── manifest.json    # SMAPI Mod 清单（必需）
├── config.json      # 默认配置（可选）
└── [其他文件]
```

## 从文件夹安装

适用于解压后的 Mod：

1. 进入实例设置 → **「Mod 管理」**
2. 点击 **「从文件夹安装」**
3. 选择 Mod 文件夹
4. 点击 **「选择文件夹」**
5. 等待安装完成

### 文件夹结构

```
ModFolder/
├── manifest.json
├── config.json
└── ...
```

## 安装第一个 Mod

### 方法一：从 NexusMods 安装

1. 进入实例设置 → **「自动安装」** 选项卡
2. 搜索想要的 Mod（如 "NPC Map Locations"）
3. 点击搜索结果
4. 选择版本
5. 点击 **「下载并安装」**

### 方法二：从 ZIP 文件安装

1. 下载 Mod 的 ZIP 文件
2. 将 ZIP 文件直接拖放到 SVL 窗口
3. 或者在 **「Mod 管理」** 中点击 **「从 ZIP 安装」**

### 启用 Mod

1. 进入 **「Mod 管理」** 选项卡
2. 确保 Mod 左侧的复选框已勾选
3. 查看是否有依赖提示

## 启用/禁用 Mod

### 单个 Mod

1. 进入 **「Mod 管理」** 选项卡
2. 点击 Mod 左侧的复选框
   - ✅ 勾选 = 启用
   - ⬜ 取消 = 禁用

### 批量操作

1. 按住 `Ctrl` 点击多个 Mod
2. 右键点击选中的 Mod
3. 选择 **「批量启用」** 或 **「批量禁用」**

### 禁用效果

禁用的 Mod：
- 不会被加载
- 文件仍然保留
- 可以随时重新启用

## 更新 Mod

### 检查更新

1. 进入 **「Mod 管理」** 选项卡
2. 点击 **「检查更新」**
3. SVL 会对比 NexusMods 上的最新版本

### 更新 Mod

1. 有更新的 Mod 会显示更新图标
2. 点击 **「更新」** 按钮
3. 选择要更新的版本
4. 等待下载和安装

### 自动更新

在设置中可以启用：
- **启动时检查更新** - 每次启动自动检查
- **自动更新 Mod** - 自动更新所有 Mod

## 常见问题

### Q: Mod 安装后不显示？

检查：
1. Mod 是否启用
2. manifest.json 是否存在
3. SMAPI 是否已安装

### Q: 提示缺少依赖？

1. 查看依赖提示
2. 搜索并安装缺少的依赖 Mod
3. 重新启用原 Mod

### Q: Mod 导致游戏崩溃？

1. 禁用最近安装的 Mod
2. 查看 SMAPI 日志
3. 检查 Mod 版本兼容性
