# 版本隔离

SVL 的版本隔离功能允许你同时运行多个不同配置的游戏实例，互不干扰。

## 什么是版本隔离？

版本隔离是一种让每个游戏实例拥有独立数据的技术：

- 独立的 `Mods` 文件夹
- 独立的存档数据
- 独立的 SMAPI 配置
- 独立的游戏设置

### 优点

| 优点 | 说明 |
|------|------|
| **多 Mod 配置** | 同时维护多套 Mod 配置 |
| **存档隔离** | 不同实例的存档互不影响 |
| **避免冲突** | Mod 之间不会互相干扰 |
| **安全测试** | 测试新 Mod 不影响主配置 |

## 工作原理

### 目录结构

```
游戏基础路径/
├── Stardew Valley/           # 原版游戏文件
│   ├── Content/
│   └── ...
│
├── 实例1/                    # 隔离实例 1
│   ├── Mods/                 # 独立 Mods 文件夹
│   ├── save/                 # 独立存档
│   └── instance.json         # 实例配置
│
└── 实例2/                    # 隔离实例 2
    ├── Mods/
    ├── save/
    └── instance.json
```

### 文件链接

SVL 使用文件链接技术：

- 游戏核心文件（`Content/`）通过链接共享
- 每个实例只需要几 MB 的配置空间
- 节省磁盘空间

## 启用版本隔离

### 新建实例时

1. 创建新实例
2. 勾选 **「启用版本隔离」**
3. 选择隔离选项

### 已有实例

1. 进入实例设置 → **「设置」**
2. 启用 **「实例隔离」**
3. 选择要隔离的内容

## 隔离范围

### 可隔离内容

| 内容 | 默认 | 说明 |
|------|------|------|
| **Mods** | ✅ | Mod 文件夹 |
| **存档** | ✅ | 游戏存档 |
| **SMAPI 配置** | ✅ | SMAPI 设置文件 |
| **游戏配置** | ⚪ | `prefs` 等配置 |

### 不可隔离内容

- 游戏核心文件（`Content/`）
- SMAPI 核心文件
- 游戏可执行文件

## 数据迁移

### 从非隔离迁移到隔离

1. 启用实例隔离
2. SVL 会提示迁移现有数据
3. 选择要迁移的内容
4. 等待迁移完成

### 从隔离迁移到非隔离

1. 禁用实例隔离
2. 选择是否保留隔离数据
3. 确认操作

> ⚠️ **警告**：此操作可能导致数据丢失，请提前备份。
