## 什么是游戏实例？

游戏实例是一个独立的游戏环境，包含：

- 独立的 `Mods` 文件夹
- 独立的存档数据
- 独立的 SMAPI 配置
- 独立的游戏设置

这让你可以：
- 为不同的 Mod 组合创建不同实例
- 同时保留多个版本的 Mod 配置
- 避免不同 Mod 之间的冲突

## 创建实例

### 方法一：从主页创建

1. 点击主页左侧的 **「+」** 按钮
2. 选择 **「创建新实例」**
3. 输入实例名称
4. 选择游戏基础路径
5. 点击 **「创建」**

### 方法二：导入整合包

1. 将整合包文件（`.zip` / `.svlmodpack`）拖放到启动器窗口
2. 在弹出的对话框中输入实例名称
3. 等待安装完成

### 方法三：克隆现有实例

1. 右键点击现有实例卡片
2. 选择 **「克隆实例」**
3. 输入新实例名称
4. 等待克隆完成

## 实例概览

进入实例设置页面，你可以看到以下选项卡：

| 选项卡 | 功能 |
|--------|------|
| **概览** | 查看实例基本信息、Mod 数量、最后游玩时间 |
| **自动安装** | 从 NexusMods / CurseForge 搜索并安装 Mod |
| **设置** | 配置实例名称、描述、图标等 |
| **Mod 管理** | 启用/禁用/卸载 Mod，查看依赖关系 |
| **导出** | 将当前实例导出为整合包 |

## 实例卡片

实例卡片显示以下信息：

- **实例名称** - 自定义的实例名称
- **Mod 数量** - 当前安装的 Mod 数量
- **SMAPI 状态** - 是否已安装 SMAPI
- **图标/Logo** - 自定义实例图标
- **标签** - 收藏、最近游玩等标记

### 卡片类型

| 类型 | 说明 |
|------|------|
| 🌟 **收藏** | 已添加到收藏的实例 |
| 🕐 **最近** | 最近游玩过的实例 |
| ✨ **新建** | 新创建的实例 |

## 实例操作

### 启动游戏

- 点击实例卡片上的 **「启动」** 按钮
- 或双击实例卡片

### 快速操作

右键点击实例卡片，可以：

- **启动游戏** - 启动此实例
- **打开文件夹** - 打开实例目录
- **克隆实例** - 创建副本
- **导出整合包** - 导出为 `.svlmodpack`
- **删除实例** - 删除此实例（游戏文件不会删除）

## 实例隔离

SVL 支持实例隔离功能，确保不同实例之间的数据完全独立。

### 隔离内容

- `Mods/` - Mod 文件夹
- `save/` - 存档文件夹
- `.smapi` - SMAPI 配置

### 启用隔离

1. 进入实例设置页面
2. 切换到 **「设置」** 选项卡
3. 启用 **「实例隔离」** 选项

> ⚠️ **注意**：启用隔离后，存档数据将存储在实例目录中，不会影响游戏原生存档。
